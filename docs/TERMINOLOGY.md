# XAIPT Terminology

## Core Terms

| Term | Meaning |
|---|---|
| HOLD | Runtime pause before irreversible execution. |
| Review | Human/device authority verification stage. |
| Authority | Trusted decision control before finality. |
| Delay Window | Controlled time friction before consequence. |
| Audit Chain | Visible record of runtime state transitions. |
| Bounded Execution | Execution remains contained until authority is resolved. |
| Trust Runtime | Environment where authority, risk, and visibility shape execution. |
| Governance Surface | Public-visible control layer for understanding system behavior. |

## Canonical Sentence

XAIPT explores execution governance systems for irreversible digital actions through runtime authority, HOLD-before-finality, and audit-visible control.
