# XAIPT Public Boundary

## Public-Safe Layer

Allowed public materials:

- runtime governance diagrams,
- conceptual HOLD flow,
- authority topology diagrams,
- public-safe demo UI,
- audit visibility examples,
- high-level research notes.

## Private / Excluded Layer

Not included in public repositories:

- production transaction systems,
- financial integrations,
- anti-bypass internals,
- sovereign enforcement layers,
- private vault logic,
- implementation-sensitive authority logic,
- XLP / UNI-OS mechanisms.

## Public Claim Discipline

XAIPT should be described as:

> execution governance and authority-runtime research.

Avoid describing it as:

- a banking system,
- a payment processor,
- an RBI product,
- an anti-fraud product,
- a replacement for bank controls,
- or a production enforcement platform.
